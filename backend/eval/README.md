# RAG 评估系统

这个评估系统用于全面评估旅行攻略 RAG 系统的性能，不仅仅是传统的 TopK 和 MRR。

## 运行评估

```bash
cd backend
python scripts/evaluate_rag_retrieval.py
```

### 可选参数

- `--cases`: 指定评估案例的 JSON 文件路径（默认：`eval/rag_eval_cases.json`）
- `--json-report`: 指定保存完整 JSON 报告的路径

示例：
```bash
python scripts/evaluate_rag_retrieval.py --json-report eval/last_report.json
```

## 评估指标

### 📊 传统指标
- **Top1 命中率**：第一个结果标题包含预期关键词的案例占比
- **TopK 命中率**：前 K 个结果中任一个包含预期关键词的案例占比
- **MRR**：平均倒数排名（Mean Reciprocal Rank）

### 🎯 内容质量指标
- **必需关键词覆盖率**：检索内容包含的必需关键词比例
- **用户偏好匹配度**：检索内容与用户偏好相关度

### 🔍 质量控制指标
- **噪声率**：包含噪声标题的结果比例（如"文档开头"）
- **跨城市污染率**：包含其他城市内容的结果比例
- **禁用内容检测**：是否包含已关闭、维修等不适合推荐的内容

### ⚡ 性能指标
- **平均延迟**：每次检索的平均响应时间（毫秒）

## 评估案例

在 `eval/rag_eval_cases.json` 中定义了涵盖多个城市和不同场景的评估案例，包括：
- 自然风景+拍照+美食
- 古镇+文化体验+慢节奏
- 美食+预算+小吃
- 熊猫+美食+休闲
- 历史+文化+古迹
- 海岛+文艺+美食
- 海滩+度假+放松

每个案例都包含：
- 目的地
- 用户偏好
- 速度要求
- 特别备注
- 预期标题关键词
- 必需内容关键词
- 用户偏好匹配关键词
- 禁用内容关键词

## 状态等级

- ✅ **PASSED**: RAG 系统运行良好
- ⚠️ **WARNING**: 某些方面需要改进
- ❌ **FAILED**: 发现严重问题

## 添加新的评估案例

1. 在 `eval/rag_eval_cases.json` 中添加新的案例对象
2. 遵循现有的格式
3. 运行评估验证
